## --------ยืนยันตัวตนเข้าใช้งาน ThaID------------
จัดอยู่ในหมวด Logical / Technical Controls (การควบคุมทางตรรกะหรือเทคโนโลยี)
ซึ่งเป็นส่วนหนึ่งของ Access Control (การควบคุมการเข้าถึงระบบ) 

เหตุผลเพราะ:
1. เป็นการป้องกันการเข้าถึงก่อนเกิดเหตุ (Preventive) การใส่รหัสผ่าน (PIN), การยืนยันตัวตนด้วยชีวมิติ เช่น ลายนิ้วมือหรือใบหน้า เป็นการป้องกันไม่ให้ผู้ที่ไม่มีสิทธิ์เข้าใช้งานแอปพลิเคชันตั้งแต่ขั้นตอนแรก

2. ใช้เทคโนโลยีและซอฟต์แวร์เป็นหลัก (Technical Control)   การยืนยันตัวตนดำเนินการผ่านระบบของแอปและอุปกรณ์อิเล็กทรอนิกส์ ไม่เกี่ยวข้องกับอุปกรณ์ทางกายภาพ เช่น ประตูหรือกุญแจ และไม่ใช่นโยบายหรือระเบียบข้อบังคับ จึงไม่ใช่ Administrative Control

3. ช่วยลดความเสี่ยงจากการแอบอ้างตัวตน
ทำให้มั่นใจว่าผู้ที่เข้าใช้งานเป็นเจ้าของข้อมูลจริงป้องกันการเข้าถึงข้อมูลส่วนบุคคลโดยไม่ได้รับอนุญาต

4. เสริมความมั่นคงปลอดภัยของข้อมูลภายในระบบ
ข้อมูลสำคัญ เช่น ข้อมูลบัตรประชาชนหรือข้อมูลภาครัฐ
จะถูกเข้าถึงได้เฉพาะผู้ใช้ที่ผ่านการยืนยันตัวตนเท่านั้น
